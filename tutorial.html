1.2 Compilation

To compile a progamm : gcc -g -Wall [PROG1.c] [PROG2.c] The out put will be a.out, that will be an executable.

To change the name of the executable:

gcc -g -Wall [PROG.c] -o [fileName]

Then an executable fileName will be the output.

If a program use a function used in an other program, they have to be compiled together. In that case if the program containing the function do not contain a main function it is not a problem for the compilation.

With #ifdef ... #endif it is possible to test a fonction using -DTEST during the compilation. main() is included between these two markers, then it is possible to verify that a part of a program is working well. gcc -g -Wall -DTEST [FILE]

6.6 redirection (>, >>, 2>&1, 2>>)

  • [command] > [file] transfer results of the command to file, create it if it doesn't exist, errase it if it exists.
  • [command] >> [file] transfer results of the command to file, create it if it doesn't exist, add them at the end of the file if it exists.
  • [command] 2>&1 [file] transfer the results and the errors to the file, create it if it doesn't exist, errase it if it exists.
  • [command] 2>> [file] transfer the error only the file, create it if it doesn't exist, add them at the end of the file if it exists.

9.2 Header

orgmode file can be indicated in the extension .org, by: #-*- mode: org -*- at the beginning of the file. In that case any type of file will be considered as an orgmode file, but if the file is already .org it is not necessary to indicate it. It may interfer at some points, for example with the initial visibility options below.

9.2.1 #+STARTUP (initial visibility)

Indicate how file should be presented when it opens. 3 mains options are possible: overview, content, showall. Should start at the begining of the file with #+STARTUP: overview. Not necessary to specify it, if the file is already in .org, and if it as already been setup in the config file, as indicated below.

But because it doesn't work everytime, it may be better to configurate it directly in the configuration file with :

(setq org-startup-folded t)

An example of org-mode setting can be find here.

9.2.2 #+TODO

Allow to add different states for the TODO. Add the following on the top of the file: #+TODO: TODO(t) STARTED(s!) WAITING(w@/!) | DONE(d!) CANCELED(c@/!) C-c t on the TODO, allow to change the state In bracket is the shortcut letter and addtionnal features:

  • ! indicate the date and time when the state is choose
  • @ ask for comment when the state is choose
  • / add more than one feature

9.2.3 #+SETUPFILE

SETUPFILE allow to configure the export to html or latex for example. For the moment two SETUPFILE have been copied from internet:

  • alex-standard-latex.org used for exporting org to latex.
  • setup.config used for html export that look like Sphinx.

So at the top of your page indicate :

#+SETUPFILE: ~/org/setup.config for html. In this case src file is also required.

or

#+SETUPFILE: ~/org/alex-standard-latex.org for latex.
